Module d’apprentissage dédié à la prise en main de GitHub : gestion de dépôts, versioning, collaboration (branches, pull requests), et bonnes pratiques de développement.

## 📚 Informations générales
- **Formateur :** Mr. TOUIL  
- **Formation :** Préparation Mastère Cybersécurité – IPSSI  
- **Durée :** Module de 48 heures  
- **Objectif :** Maîtriser les bases et bonnes pratiques de Git et GitHub pour travailler en équipe sur des projets.

## 🎯 Objectifs du module
- Comprendre les bases de Git (commit, push, pull)
- Gérer un dépôt GitHub
- Travailler avec les branches
- Utiliser les pull requests
- Collaborer sur un projet en équipe
- Appliquer des bonnes pratiques de versioning

## 🛠️ Technologies utilisées
- Git
- GitHub
- Terminal
- VS Code

## 📁 Organisation du projet
- `main` : branche principale
- branches de features pour les développements
- issues pour la gestion des tâches
- milestones pour le suivi des versions

## 👥 Collaboration
- Travail en équipe de 2
- Assignation des issues
- Revues de code via pull requests

## 🚀 Bonnes pratiques
- Commits clairs et explicites
- Une branche par fonctionnalité
- Utilisation des labels et issues
- Code propre et documenté
